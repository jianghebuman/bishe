# 一键更新部署说明

这套流程用于 `bishe.ryvyn.online` 线上更新。服务器内存较小，不在服务器构建前端；本地构建，服务器只部署产物。

## 第一次准备

服务器只需要安装一次部署脚本：

```bash
cd /www/campus-recruitment-src
git pull
mkdir -p /www/campus-recruitment/scripts
cp deploy/linux/deploy-bishe-release.sh /www/campus-recruitment/scripts/deploy-bishe-release.sh
chmod +x /www/campus-recruitment/scripts/deploy-bishe-release.sh
```

以后不用重复这一步，除非部署脚本本身更新了。

## 以后每次更新代码

在本地 PowerShell 进入项目根目录：

```powershell
cd "C:\Users\Administrator\Desktop\基于SpringBoot框架的校园求职招聘系统设计与实现"
.\deploy\windows\publish-bishe.ps1
```

脚本会自动完成：

- 构建后端 `campus-recruitment.jar`
- 构建前端 `dist`
- 写入 `version.txt`
- 打包 `deploy/releases/bishe-release-时间-commit.zip`
- 把 `数据库脚本/update-data.sql` 和 `upload` 演示素材放进发布包

然后用 FinalShell 把生成的 zip 上传到服务器，例如：

```text
/www/releases/bishe-release-xxxx.zip
```

如果只更新前端或后端，不更新数据库：

```bash
/www/campus-recruitment/scripts/deploy-bishe-release.sh /www/releases/bishe-release-xxxx.zip
```

如果这次包含数据库结构或演示数据更新：

```bash
/www/campus-recruitment/scripts/deploy-bishe-release.sh /www/releases/bishe-release-xxxx.zip --db
```

`--db` 会先备份线上库到：

```text
/www/campus-recruitment/backups/
```

然后执行发布包里的：

```text
database/update-data.sql
```

## 验证

服务器执行完成后检查：

```bash
curl http://127.0.0.1:8081/api/public/home
curl "https://bishe.ryvyn.online/version.txt?ts=$(date +%s)"
```

浏览器按：

```text
Ctrl + F5
```

## 数据库更新规则

- 线上不要导入整份 `数据库脚本/campus_recruitment.sql`
- 线上只执行 `数据库脚本/update-data.sql`
- `update-data.sql` 必须是增量脚本，保留线上用户自己注册的账号和业务数据
- 删除、修改演示数据时，写成明确的 `UPDATE` / `DELETE` / `INSERT ... ON DUPLICATE KEY UPDATE`
- 兼容服务器 MariaDB，尽量不要使用存储过程
- 执行数据库更新前必须备份

以后让 Codex 更新数据库时，可以直接说：

```text
按 bishe 线上部署规则更新数据库：只改 数据库脚本/update-data.sql，做增量迁移，保护线上用户注册数据，兼容 MariaDB，不要让我重新导入 campus_recruitment.sql。更新完成后按一键部署脚本打包上传。
```
