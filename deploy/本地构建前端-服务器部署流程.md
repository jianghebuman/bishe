# 本地构建前端，服务器仅部署产物

适用场景：服务器内存较小（例如 2G），在服务器上执行 `vite build` 容易被系统杀掉。

当前项目推荐的稳定部署方式：

1. 本地电脑构建前端 `dist`
2. 本地电脑构建后端 `jar`
3. 上传前端 `dist` 和后端 `jar` 到服务器
4. 服务器只负责替换产物、重启后端、重载 Nginx

## 一次更新的推荐流程

### 1. 本地更新代码并构建后端

```bash
cd 项目源码/基于SpringBoot框架的校园求职招聘系统设计与实现/backend
mvn clean package -DskipTests
```

构建完成后，产物为：

```text
target/campus-recruitment.jar
```

### 2. 本地构建前端

```bash
cd 项目源码/基于SpringBoot框架的校园求职招聘系统设计与实现/frontend
npm install
npm run build
```

构建完成后，产物目录为：

```text
dist/
```

### 3. 上传产物到服务器

服务器当前运行目录：

```text
/www/campus-recruitment/
├── backend/campus-recruitment.jar
├── frontend/dist/
├── upload/
├── logs/
└── run/
```

需要上传替换：

```text
backend/target/campus-recruitment.jar  ->  /www/campus-recruitment/backend/campus-recruitment.jar
frontend/dist/*                        ->  /www/campus-recruitment/frontend/dist/
```

如果使用 FinalShell：

- 左侧文件面板打开本地项目目录
- 右侧文件面板打开服务器 `/www/campus-recruitment/`
- 将本地 `backend/target/campus-recruitment.jar` 拖到服务器 `backend/`
- 将本地 `frontend/dist/` 整个目录内容覆盖到服务器 `frontend/dist/`

### 4. 服务器重启后端

```bash
if [ -f /www/campus-recruitment/run/campus-recruitment.pid ]; then
  kill $(cat /www/campus-recruitment/run/campus-recruitment.pid) 2>/dev/null || true
  rm -f /www/campus-recruitment/run/campus-recruitment.pid
fi

APP_HOME=/www/campus-recruitment /www/campus-recruitment/scripts/start-campus-recruitment.sh
```

### 5. 重载 Nginx

```bash
/www/server/nginx/sbin/nginx -t
/www/server/nginx/sbin/nginx -s reload
```

### 6. 验证

```bash
curl http://127.0.0.1:8081/api/public/home
curl https://bishe.ryvyn.online/version.txt?ts=$(date +%s)
```

浏览器建议：

```text
Ctrl + F5
```

## 注意事项

- 不要重新导入整份 `数据库脚本/campus_recruitment.sql`
- 不要覆盖服务器 `/www/campus-recruitment/.env`
- 数据库结构或种子数据更新时，单独执行增量脚本：

```bash
mysql -uroot -p12345678 campus_recruitment < "数据库脚本/update-data.sql"
```

- 项目自带演示图片若有变化，需要同步：

```text
项目源码/.../upload/image/seed/*
```

到服务器：

```text
/www/campus-recruitment/upload/image/seed/
```

## 为什么推荐这种方式

- 服务器 2G 内存不适合频繁运行 `vite build`
- 服务器运行静态前端文件本身几乎不吃内存
- 本地构建成功率高，部署流程更稳定
